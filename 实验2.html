网页程序设计

前言

前言： 万维网是一个庞大的浩瀚的资源库，信息库，HTML 超链接是各个 web 资源之间的纽 带。超链接是当前 web 成功的一个关键。
超链接有两个端点和一个方向：链接开始与“源”端并指向“终”端。从源到终的指 向就是超链接的方向。源端存在于定义链接的文档中，终端则通过一个链接属性来指定。
通常，一个终端指向一个特定的 web 资源，如一个 HTML 文档、一个图像、一个影音 文件、一段声音等等，也可以指向当前文档。

1 两类链接

link 元素只能出现在 HTML 文档的头部；它定义了当前文档与另一个资源之间的联系，尽 管 link 没有内容显示，而它定义的联系依然会被某些用户浏览器渲染。
a 元素只能出现在 HTML 文档的 body 部分；它定义了当前文档中某个区域和另一个资源之 间的联系。a 元素的内容（文本、图像等）将本浏览器呈现，并且浏览器通常会突出显示这 个内容来指出连接的存在。

2 两类路径

每个网页作为都有一个唯一的地址，称作为统一资源定位符（URI） 绝对路径：URI 的完整路径称为绝对路径，是文档在 WWW 空间上的唯一定位地址； 2 相对路径：省略掉当前文档和所链接的文档的 URI 中前面的相同部分，就构成了相对路径。 相对路径优点：简单；便于网站迁移。 绝对路径：parser

3 三个区分

URI（Uniform Resource Identifier，统一资源标识符） 定位任何远程或本地的可用资源
URL（Uniform Resource Locator，统一资源定位符） URI 的子集
URN（Uniform Resource Name，统一资源名称） URI 的另一个子集。
NID（Namespace Identifier，命名空间标识）
NSS（Namespace Specific String，命名空间具体字符串） urn: :
urn:isbn:9867529995
urn:uuid:6e834430-9v3a-lld9-9969-012334466

12.2使用link元素说明文档集合之间的关系

Start

表明链接指向文档所在文档集合中的起始文档

Next

表明链接指向当前文档所在文档集合中的下一个文档。浏览器可以根据该属性的值，预加载 下一个文档

Contents

表明链接指向的文档是一个内容表，也就是目录。一些浏览器可以使用 TOC（table of Contents）实现列表功能。

Index

表明链接指向的问答是为当前文档提供索引的一个文档

Glossary

表明链接指向的文档是附属于当前文档、而为文档中的术语提供说明的文档

Chapter

表明链接指向的文档是当前文档所在的文档集合中的一章内容

Section

表明链接指向的文档是当前文档所在的文档集合中的一段文本

Subsection

表明链接指向的文档是当前文档所在的文档集合中的一段文本的节选

Appendix
